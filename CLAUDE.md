# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Structure

Monorepo with a React frontend and multiple backend implementations sharing a single OpenAPI spec:

```
backend/
  openapi.yml               # Source of truth for all API contracts and schemas
  openapi-express-ts/       # Active backend (Express v5 + OpenAPI-backend + Prisma + MongoDB)
  nodejs-express-ts/        # Alternative backend (Express v4, manual routing)
frontend/                   # React 19 + Vite + TailwindCSS v4 SPA
```

The `backend/openapi-express-ts/` directory has its own `CLAUDE.md` with deeper backend-specific guidance.

## Commands

### Backend (`backend/openapi-express-ts/`)
```bash
pnpm start:dev       # Dev server with hot reload (tsx watch)
pnpm build           # Bundle with esbuild → dist/index.js
pnpm check:type      # TypeScript type checking
pnpm generate:prisma # Regenerate Prisma client after schema changes
```

### Frontend (`frontend/`)
```bash
pnpm dev             # Vite dev server (MSW mocks enabled in dev)
pnpm build           # tsc + vite build
pnpm test            # Vitest (jsdom environment)
pnpm test:ui         # Vitest with UI and coverage
pnpm lint            # ESLint
pnpm check:types     # TypeScript check (no emit)
pnpm check:format    # Prettier check
pnpm check:dead-code # Knip dead code analysis
pnpm gen:sdk         # Regenerate SDK from openapi.yml via orval
pnpm storybook       # Storybook dev server on port 6006
```

### Root
```bash
docker compose up    # Swagger Editor (:8081), Swagger UI (:8082), frontend (:3000)
git cz               # Commitizen for conventional commits
```

## OpenAPI as Source of Truth

`backend/openapi.yml` drives everything:
1. **Backend**: `openapi-backend` routes requests by `operationId` and validates against the spec
2. **Frontend SDK**: `orval` generates typed axios+react-query hooks into `frontend/src/sdk/generated/`
3. **Zod schemas**: also generated by orval into `frontend/src/sdk/generated/*.zod.ts`

**Workflow for API changes**: update `openapi.yml` → run `pnpm generate:prisma` in backend → run `pnpm gen:sdk` in frontend.

## Frontend Architecture

**Path aliases** (defined in `vite.config.ts`):

| Alias | Path |
|-------|------|
| `@Auth` | `src/Auth` — auth context, login pages, JWT handling |
| `@Sdk` | `src/sdk/generated` — orval-generated API client |
| `@Common` | `src/Common` — shared utilities, context factory |
| `@Teams`, `@Player`, `@Game`, `@Calendar`, `@Settings` | Feature modules |
| `@Layouts` | `src/Layouts` — route layout shells |
| `@I18n` | `src/I18n` — react-intl setup and locale files |
| `@Config` | `config/` — axios instance, react-query client |

**Key patterns**:
- **State management**: React context via `createContextWithWrite` (context + `useReducer` factory in `@Common/Context/`)
- **Server state**: TanStack Query v5 with orval-generated hooks from `@Sdk`
- **Auth**: JWT stored in localStorage, read by `useCheckAuthLocalStorage`. `CheckAuthentication` guard wraps protected routes.
- **Mocking**: MSW (Mock Service Worker) is enabled in non-production builds. Mock handlers are auto-generated by orval alongside the SDK.
- **Routing**: React Router v7, file-based feature modules. All routes defined in `AppRouting.tsx`.
- **Forms**: react-hook-form + zod resolvers using generated zod schemas.
- **i18n**: react-intl with locale files under `src/I18n/locales/`.
